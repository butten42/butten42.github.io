react 初探

发表于 2019-11-13 | | 阅读次数:
字数统计: 496字 | 阅读时长 ≈ 1分钟

react是我学习的第四个前端框架，顺序依次是jquery,knockout,vue,react。看过了官方教程&文档，在fcc上看了一个react markdown项目，算是对react有了一个初步的认识。

对于vue玩家来说，react有它的优点，但缺点也十分明显，没有MVC，我初看的时候还以为是面条编程，而事实上react也确实更偏向于函数式编程，而vue则更看重对象，vue给玩家的选择并不多，玩家也不需要多关心vue内部的事情，而react的自定义空间更大，玩家的参与度更高。

我目前还是比较习惯vue。

例子

官方从并子棋游戏开始了解react特性，props传递，状态提升，事件，函数式组件和class组件。

除了一个边做边学的例子外还有对应的文档，老实说我觉得文档不怎么样，翻译的也有些拗口，外国人就是不会教人。

核心概念有：

  • JSX
  • 元素渲染
  • 组件&props
  • state & 生命周期
  • 事件处理
  • 条件渲染
  • 列表&key
  • 表单
  • 状态提升
  • 组合VS继承
  • react 哲学

最后一章是react的核心设计思想，分层，单一功能原则，ui和js的合一性。但是这些核心之外还有hook，和reactDOM等等要继续学习，我不知道是否要继续，无论是vue还是react都没有什么太大的差别，我把vue学好学精来应该就能应对大部分的工作了，而且vue从react身上也学习了状态管理，总体趋势应该是大家殊途同归。

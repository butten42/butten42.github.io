一些关于腾讯微云的改造经验

发表于 2019-11-22 | | 阅读次数:
字数统计: 375字 | 阅读时长 ≈ 1分钟

写了一个作用于微云网页版的连续下载脚本，用处是在设定的时间内（1分钟？5秒钟？）顺序下载文件夹内的所有文件。

为什么要这样做

因为微云网页版的限制太多了，打包下载很容易超出总文件大小限制，而且速度不如单个下载，手动点击的话一是太麻烦，二来点击的过于频繁是不能触发下载的，加上网页版疑似有同时下载个数的限制，超过了很容易下载坏档。

我写的脚本设定为每1分钟执行一次顺序下载，我也试过设定为5分钟但不知道怎么回事脚本没有执行，所以不建议设定的太久。

原理

大致思想就是获取文件夹内的下载条目，遍历，对应每隔一段时间执行一次模拟的点击下载。

这里的难点在于在在循环里的延时操作，可以用生成器yield保留每次循环，在定时器里每次取出就可以了。

一些tips

我本来想用Promise封装setTimeout做sleep方法，直接在每次循环里调用，但并没用——当然没用。

微云有$对象方法，我本来以为就是jq里的方法，但并非如此，看来只是微云自己封装的一个全局对象，包含一部分dom操作，但和jq还是不一样的。
